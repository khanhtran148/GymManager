# Full Codebase Review — 2026-03-19

## Executive Summary

Four parallel concern agents reviewed the entire GymManager codebase (backend + frontend). The review identified **6 CRITICAL**, **20 HIGH**, **14 MEDIUM**, and **7 LOW** findings across security, quality, performance, and test coverage.

**Top 3 systemic risks:**
1. **Cross-tenant IDOR** — Subscription/member lookups by ID bypass gym-house scoping, enabling cross-tenant data manipulation
2. **28 of 59 handlers untested** — Nearly half the Application layer has zero test coverage; all 29 validators lack dedicated tests
3. **Client-controlled auth cookies** — `is_authenticated`, `user_role`, and `route_access` cookies are writable by JavaScript, undermining frontend route guards

## Recommended Fix Priority

1. **Cross-tenant IDOR** (S12, S13) — Add `GymHouseId` validation after entity retrieval in subscription handlers
2. **Secrets in source control** (S1, S2) — Move to environment variables / secrets manager
3. **ApiControllerBase string matching** (Q1) — Replace with strongly-typed error union
4. **N+1 queries** (P1, P2, P3) — Batch DB operations in loops
5. **Test coverage gaps** (T1, T2) — Add validator unit tests and handler coverage for 28 missing handlers
6. **Token storage** (S3) — Move refresh token to HttpOnly cookie
7. **Client-controlled auth cookies** (S4, S5) — Remove or make server-set HttpOnly
8. **PermissionChecker interface honesty** (S9) — Remove unused params or implement real lookup
